Example Class - Liquor Bottles

Classes are a powerful tool in Ruby - both because the built-in classes contain a tremendous amount of useful methods, and because we can build our own classes, complete with their own dedicated methods!

But when do we want to create a class of our own, instead of simply using the methods we've already learned? To me, the answer lies (as it so often does), in the principle of DRY - Don't Repeat Yourself.

We can try to think of Classes as representative of reality - a car is an instance of a class of automobile model, which is a subclass of a brand of cars, which are a subclass of vehicles, and so on - but this kind of thinking will always run up against limits. No programming language can perfectly represent reality, nor should it - the programs we write will have specific tasks to accomplish, and it is best to think of the code as being in service of those tasks (narrow, focused, accomplishable goals), rather than trying to mimic the universe (expansive, overreaching, complex systems).

Let's use an example. Say we're writing code as part of an inventory control system for bar managers. These people have to keep track of hundreds of different types and brands of alcohol, all of which come in various bottles. Here's how we might write a class that each bottling can be an instance of:

class Liquor_Bottle

attr_reader :size, :name, :category, :size_unit, :ounce_size

  def initialize(name, category, size, size_unit)
    @acceptable_sizes = ["ml", "oz", "L"]
    @name = name
    @category = category
    @size = size
    @size_unit = size_unit
    raise ArgumentError.new("Not an approved size unit")
    unless @acceptable_sizes.include?(@size_unit)
    set_ounce_size
  end

  def set_ounce_size
    case @size_unit
    when "ml"
      @ounce_size = size * 0.033814
    when "L"
      @ounce_size = size * 0.000033814
    else
      @ounce_size = size
    end
  end

  def describe
    puts "#{name} #{category} - #{size}#{size_unit}"
    puts "(#{ounce_size} ounces)" if size_unit != "oz"
  end
end


bulleit = Liquor_Bottle.new("Bulleit", "Bourbon", 750, "ml") bulleit.describe

The manager might want to also know what each size means in a common measurement, so we included code to translate ml and liters into ounces.

This is just one example, but it should give the reader an idea what a class is used for. Bottles have many properties that exist in reality that are irrelevant here - the density of the glass, the shape of the neck, the color of the label - but the class exists to serve the end user, who cares about how to organize and order alcohol, and what it costs per ounce.
